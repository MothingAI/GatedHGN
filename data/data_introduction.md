## 📈 图结构特点

### 异构图特性

```
✓ 多种节点类型 (company, industry)
✓ 多种边类型 (4种不同的关系)
✓ 语义丰富 (风险溢出、归属、包含、供应链)
✓ 元路径多样性 (可定义多条语义路径)
```

### 网络特征

| 网络 | 密度 | 特征 |
|------|------|------|
| 企业网络 | 51.2% | 非常密集 |
| 行业网络 | 4.2% | 相对稀疏 |
| 企业-行业 | 中等 | 平均1.67条边/企业 |

### 元路径定义

基于这个图结构，模型定义了3条元路径用于风险识别：

#### 1. Direct (直接路径)
```
路径: company → company
边类型: spillover
语义: 直接风险溢出
应用: 捕捉企业间的直接风险传导
```

#### 2. Industry Mediated (行业中介)
```
路径: company → industry → company
边类型: belongs_to + contains
语义: 通过行业传导的风险
应用: 捕捉同行业企业间的风险溢出效应
```

#### 3. Supply Chain (供应链)
```
路径: company → industry → industry → company
边类型: belongs_to + contains + supply_chain
语义: 通过产业链传导的风险
应用: 捕捉供应链上下游的风险传导
```

## 💾 数据存储结构

### HeteroData对象结构

```python
HeteroData {
    # ============ 节点数据 ============

    'company': {
        'x': Tensor([567, 10]),      # 企业特征矩阵
        'y': Tensor([567]),          # 企业风险标签
        'node_id': Tensor([567])     # 企业节点ID
    },

    'industry': {
        'x': Tensor([45, 23]),       # 行业特征矩阵
        'node_id': Tensor([45])      # 行业节点ID
    },

    # ============ 边数据 ============

    ('company', 'spillover', 'company'): {
        'edge_index': Tensor([2, 164351])  # [源节点, 目标节点]
    },

    ('company', 'belongs_to', 'industry'): {
        'edge_index': Tensor([2, 945])
    },

    ('industry', 'contains', 'company'): {
        'edge_index': Tensor([2, 945])
    },

    ('industry', 'supply_chain', 'industry'): {
        'edge_index': Tensor([2, 83])
    }
}
```
